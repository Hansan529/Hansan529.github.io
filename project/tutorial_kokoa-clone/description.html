#3.12 Fixed

'position'은 레이아웃보다는 위치 조정이 조금 필요한 상황에서 사용한다.

그 중에 fixed라는 속성이 있는데 이는 해당 태그를 고정시키기 위해 사용된다.

div {
width:50px;
height:50px;
background-color:black;
}

.accept {
position: fixed;
width:60px;
background-color:green;
}

해당 accept 클래스를 갖은 태그가 화면에 고정된다. 단 해당 태그가 초기에 위치한 좌표에 고정이된다.

<div>1번</div>
<div class="fixed">고정</div>

div 위치에 50x50px black "1번" 박스가 생성되고, 그 밑에 60x50px green "고정" 박스가 생성된다.

해당하는 위치에 fixed가 된다. 이를 방지하기 위해서는 아래와 같은 스타일을 추가로 적용하면 된다.

top: @px;
right: @px;
bottom: @px;
left: @px;
4가지 중 단 1가지라도 변경

별도의 위치를 설정만 해도 기존 1번 div는 무시하고 상단에서 @px, … 만큼 떨어진 채로 이동된다.

정리하면 fixed를 적용한 순간부터 원래 있던 위치에 고정되며 서로 다른 레이어가 되어 간섭이 되지 않도록 바뀐다. 그 다음 위치를 내가 원하는 위치로 이동시킨다.

#3.13 Relative Absolute

position의 default 값은 static이다.

'static'은 레이아웃이 박스를 처음 위치하는 곳에 두는 것을 말한다.

.box{ position: relative; left: 20px; background-color: blue; }

<div></div>
<div class="box"></div>

'relative'는 개체가 기존에 있던 곳을 기준으로 이동한다. 적은 위치 변경이 필요할 때 사용한다.

.box{ position: absolute; right: 100px; bottom: -10px; background-color: blue;

<div></div>
<div class="box"></div>

'absolute'는 가장 가까운 부모(static이 아닌)를 기준으로 위치를 이동한다. 부모의 스타일을 적용받지 않게 된다.

부모의 position이 static일 경우에는 [초기 컨테이닝 블록 = viewport = 현재 화면 영역의 크기]가 기준이 되므로 부모를 static이 아닌 다른 속성으로 설정하면 부모의 위치를 기준으로 변동한다.
